Git学习

Posted on 2018-06-01 | In git

今天学习一下简单的git语法在这里做一下简单的总结，作为将来复习的依据。git commit的作用是在原有的基础将新加入的修改记录下来并生成更新后的目录。可以在改动时注明改动的原因可以做为查找时候的依据，我为了在结构对这样的语法有一个深入的认识现在就是做一些简单的总结，工作区就是我们自己建立的一个文件夹，在这个文件夹内有一个后缀为.git的文件，这个.git就是我们要找的版本库（repository）其中有一个stage称为暂存区或者index可以将工作区的文件通过git add添加到到版本库的暂存区，暂存区中文件可以通过使用git commit加入分支，一般默认建立的分支名称为master,我们来将这个过程分为几步来完成：

1、建立版本库

该如何建立版本库呢？

我们可以通过在自己建立的文件夹中使用git init在文件夹中创建一个.git结尾的文件，这就是我们创建的版本库，在这个版本库中包含内上述的暂存区，分支，我们新建立的文件通过使用git add+文件名都会加入到暂存区（stage/index）通过git commit 加入目录master。

2、在建立了版本库之后我们使用以下几种语法呢？

git commit 使用git commit可以推送至分支，并且可以添加注释。
git log 可以查询我们每次向分支添加的改动时间和改动的原因
git branch +分支名 可以创建新的分支
git checkout +分支名可以实现分支之间的切换
git HEAD 可以退回至上已经更改的未更改
